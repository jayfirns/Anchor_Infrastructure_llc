# Seed Stack

A ready-to-run Docker Compose setup with three services that replace cloud subscriptions with software you own.

## What's In the Box

| Service | Port | What It Does |
|---------|------|-------------|
| **Pi-hole** | 53 (DNS), 80 (admin) | Blocks ads and trackers for your entire network |
| **Vaultwarden** | 8080 | Self-hosted password manager (works with Bitwarden apps) |
| **Uptime Kuma** | 3001 | Monitors your services and alerts you when something's down |

## Requirements

- A machine running Linux (Raspberry Pi, old laptop, mini PC, VM — anything)
- 2GB+ RAM (4GB recommended)
- Docker and Docker Compose installed ([instructions](../docs/docker-basics.md))

## Quick Start

```bash
./setup.sh
```

The setup script will:
1. Verify Docker is installed and running
2. Generate passwords and create your `.env` file
3. Pull the container images
4. Start everything

That's it. Access your services at:
- **Pi-hole:** `http://YOUR_IP/admin`
- **Vaultwarden:** `http://YOUR_IP:8080`
- **Uptime Kuma:** `http://YOUR_IP:3001`

## Manual Start

If you prefer to do it yourself:

```bash
cp .env.example .env
# Edit .env — set passwords and timezone
docker compose up -d
docker compose ps
```

## After Setup

1. **Pi-hole:** Log into the admin panel and set it as your router's DNS server (check your router's DHCP settings)
2. **Vaultwarden:** Create your account, then set `SIGNUPS_ALLOWED=false` in `.env` and run `docker compose up -d`
3. **Uptime Kuma:** Add monitors for Pi-hole, Vaultwarden, and anything else you care about

## Health Check

```bash
./health-check.sh
```

Shows you what's running and what still needs attention. The warnings are intentional — they're the things that separate "I have containers" from "I have infrastructure."

## Updating

```bash
docker compose pull && docker compose up -d
```

Pulls the latest images and restarts with zero data loss (data lives in Docker volumes).

## Files

| File | Purpose |
|------|---------|
| `docker-compose.yml` | Service definitions |
| `.env.example` | Configuration template |
| `.env` | Your actual configuration (generated by setup.sh, not committed) |
| `setup.sh` | First-time setup: prereq check, secret generation, launch |
| `health-check.sh` | Service status + configuration gap analysis |
| `WHAT_THIS_IS.md` | Disclaimers, upstream links, and what this is (and isn't) |
